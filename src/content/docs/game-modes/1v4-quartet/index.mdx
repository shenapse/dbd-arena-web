## Queue requirement

Survivors are expected to queue as a full four-player party.

| Who | How they queue | Declares at queue time |
|---|---|---|
| Killer player | Queues solo. | One killer character. |
| Survivor player | Queues as a group of four. | Nothing. |

## Result facts

| Fact | Notes |
|---|---|
| Winning side | Which side won — killers or survivors. Omitted on a draw. |
| Draw | Marks a drawn match. Mutually exclusive with a winning side. |
| Generators remaining | How many generators were left unrepaired at the end (0-5) |
| Hooks | Total hooks performed by the killer during the match (0-12). |
| Kills | How many survivors were killed (0-4). |

## Ratings

1v4 Quartet Ranked uses ratings that are both game-mode specific and killer specific.

The same rating scope is used in both phases:

- **Matchmaking phase:** players are matched using their 1v4 Quartet Ranked rating for the relevant killer.
- **Rating update phase:** only that same scoped rating is updated after the match.

For a match where player K plays Nurse against survivor team S=(s1, s2, s3, s4), the ratings used and affected are:

- (K, {1v4 Quartet Ranked / role=killer / killer=Nurse})
- (S, {1v4 Quartet Ranked / role=survivor / killer=Nurse})
- **NOT** (s1, {1v4 Quartet Ranked / role=survivor / killer=Nurse})
- **NOT** (S, {some other game mode / role=survivor / killer=Nurse})
- **NOT** (S, {1v4 Quartet Ranked / role=survivor / killer=Blight})
